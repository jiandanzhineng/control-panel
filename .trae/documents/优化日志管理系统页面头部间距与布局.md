## 问题定位
- 全局样式在 `frontend/src/App.vue:125-131` 重置了所有元素的默认外边距（`* { margin: 0; }`），导致 `frontend/src/views/LogManagement.vue:3` 的 `<h1>` 与下方两个按钮区域贴得很近。
- 当前页面结构：标题独占一行，按钮在下一行（`LogManagement.vue:5-18`），但标题与按钮之间无显式间距；按钮本身 `gap: 10px` 已有，但整体观感仍偏紧。

## 改进方案
- 将标题与两个按钮收敛为一个“页面头部条”，左侧标题、右侧按钮，形成清晰的层级与空间。
- 为头部条添加合理的垂直间距与按钮间距，并提供简单的移动端折行处理。

## 具体改动
1. 更新模板（`frontend/src/views/LogManagement.vue`）
- 用一个容器包裹标题与按钮：
```
<div class="page-header">
  <h1>日志管理系统</h1>
  <div class="tabs">
    <button :class="{ active: activeTab === 'realtime' }" @click="activeTab = 'realtime'">实时日志</button>
    <button :class="{ active: activeTab === 'files' }" @click="activeTab = 'files'">历史文件</button>
  </div>
</div>
```
- 原来的独立 `<h1>` 与下方 `.tabs` 合并为上述结构，`tab-content` 保持不变。

2. 更新样式（同文件 `style scoped`，`LogManagement.vue:35-66`）
- 添加头部条与优化按钮间距：
```
.page-header { display: flex; align-items: center; justify-content: space-between; margin-bottom: 16px; }
.tabs { display: flex; gap: 12px; }
```
- 保留现有 `.tabs button` 与 `.tabs button.active` 的配色与尺寸设置。

3. 移动端适配
- 在同一 `style scoped` 中追加简单媒体查询：
```
@media (max-width: 768px) {
  .page-header { flex-direction: column; align-items: flex-start; gap: 12px; }
}
```
- 小屏下标题与按钮纵向排列，保持呼吸感。

## 验证点
- 标题与按钮之间出现清晰的空间分隔；按钮之间间距更舒适。
- Tab 切换行为保持（`activeTab` 逻辑未改动）。
- 其他组件（`RealTimeLog.vue`, `LogFileList.vue`）与面包屑头部（`App.vue`）不受影响。

## 备注
- 若更偏向极简改动，也可只为标题增加下边距：在同文件添加 `.log-management h1 { margin-bottom: 12px; }`；但头部条布局方案观感更统一、可扩展（后续可加入更多右侧操作）。
